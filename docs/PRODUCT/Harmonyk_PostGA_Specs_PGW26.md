# Harmonyk — Post-GA (6 months / PGW26) Product Specifications

| Area | What ships in 6 months | Notes / Guardrails |
|---|---|---|
| **Core Platform & Auth** | Multi-org workspaces; Supabase auth (magic links, Google SSO); org/member roles; protected AppShell; responsive UI; WCAG-AA basics. | Centralized brand assets under `public/brand/`. |
| **Vault & Document Model** | Central Vault for contracts, decks, account packs, uploads; versions; rich metadata (type, status, tags, owner, deal/contact links); activity history. | **Vault-only semantics** for embeddings/search; external docs ingested as refs → explicitly "Vaulted" to embed. |
| **Vault Hygiene & Health** | **Hygiene Agent** (Jobs/Queue) for dupes/near-dupes, bad names, untagged/stale; Review Queue to accept/ignore fixes; **Workspace Health** dashboard with one-click "Fix" entries. | Non-destructive by default; every change is auditable. |
| **Workbench & Deals** | Workbench = "My Company Today": deals snapshot, pending signatures, recent shares, key Account packs, important tasks; **Deal** entities link contracts, decks, tasks, signatures; **Deal Agent** scores hot vs stuck and proposes next actions. | **Maestro** panel is page-aware and suggests next steps. |
| **Contracts Builder** | Canonical library (Operational & HR; Corporate & Finance; Commercial & Dealmaking); clause editor; **ClauseGraph v1.5/2** (catalog + matrix/heatmap for usage/risk); clause CRUD with alternates, canonical/soft/aggressive flags, jurisdiction tags; clause-aware diff/redline import; lifecycle Draft → In review → Approved → Sent → Signed → Active/Expired. | Sign flows via Documenso + optional DocuSign/Dropbox Sign/Adobe Sign; status also surfaces in **Deals** and **Signatures Center**. |
| **Decks Builder** | Fundraising & Investor Update templates; starter Sales/Customer decks; variant decks per audience; **Missing Pieces** checker; **Visual Deck Assistant v1** (suggest/generate/insert visuals; org/user rate limits + kill-switch); **Deck Analytics** (views, last viewed, optional per-slide); Investor Feedback view. | **Accounts → Decks**: push financial blocks with refreshable links. |
| **Accounts Builder & Packs** | Financial Inbox + classifier (receipts, invoices, bank statements, accountant packs); packs: **SaaS Expenses**, **Investor Snapshot**, **Runway Planner**, **KPI Notebook**; scenario toggles (best/base/worst); deck-ready blocks (runway charts, KPI tables, SaaS spend). | Live data via **QuickBooks, Xero, Stripe** (RO) + Google Sheets (RO). |
| **Tasks & Calendar** | Task Hub (CRUD, filters, tiles, types: docs/deals/follow-ups); recurring tasks & templates (weekly investor update, monthly close, renewals); outbound sync to Asana/Linear (optional inbound); Calendar week/month views with **read-only Google Calendar mirror**; daily/weekly digests. | Write-back to Calendar can be feature-flagged later. |
| **Playbooks & Automation** | Starter Playbooks (Weekly Investor Update, Seed Fundraise Prep, New Customer Onboarding, Renewal flows); **Playbooks v2 editor** with audited action palette (search, summarise, generate doc/deck/email, create tasks, share, email/Slack/Teams); triggers: cron, e-sign events, connector webhooks; per-run audit logs; Playbook suggestions on Workbench/Maestro/Contact/Deal views. | Draft → simulate → approve → run. **No silent changes.** |
| **Maestro AI & RAG** | Maestro proposes actions and can trigger approved Playbooks; **RAG v1** (Vault-first with citations & "Why this?"); **RAG v2** federated over selected externals (Drive/Gmail + chosen connectors) with explicit provenance; golden QA sets per Builder; org-level memory (jurisdiction, tone, risk, currency/sector) with per-Playbook overrides. | Human-in-the-loop always. |
| **Share Hub, Signatures & Redaction** | DB-backed share links (create/revoke/regenerate; expiry; password; org policies); watermarked views; **redaction** for tagged sensitive blocks; tokenized links + tracking pixel; **Signatures Center** aggregating Documenso + external e-sign via webhooks + Jobs/Queue + DLQ/replay; signatures analytics → Insights/Deals/Contacts; multi-step sign flows via Playbooks. | Privacy copy on shared views; full audit trail. |
| **Contacts & Engagement** | Contacts DB (CRUD, RLS); auto-link contacts to docs/decks/share links/signatures/tasks/deals; contact timelines; engagement scoring (cold/warm/hot); optional Google Contacts import; PII flags + export/erasure pathways. | Controlled by feature flags for PII flows. |
| **Insights, Dashboards & Digests** | Metrics: docs created/shared/signed; pack runs; task completion; connector health; **Workspace Health**; funnels (contracts & decks); Deal/Investor dashboards (hot vs stuck + actions); plan-usage dashboards; weekly org digests (contracts, decks, packs, deals, tasks, runway, top contacts). | CSV/PDF export; scheduled reports. |
| **Guided Experience** | **User Progress Narrator**: `UserProgressState` → Dashboard Hero CTA + **Maestro** quick-start prompts stay in sync (onboard connectors/import/classify/start deal/enable automation/daily focus). | All CTAs instrumented; exactly one primary CTA at a time. |
| **Viral Loops** | **Free Collaborator roles** (paying workspaces): unlimited view/comment/suggest; inline upsell. **Keep a Copy in Your Vault** for recipients/signers → 1-click Free workspace. **Template "Clone for Yourself".** | Non-blocking; respectful; instrumented (`invite_collaborator`, `claim_signed_doc`, `clone_template`). |
| **Connectors (≈25)** | Email: Gmail, Outlook. Storage/Docs: Drive, OneDrive, SharePoint libs, Dropbox, Box. Decks: Slides, PowerPoint, Pitch, Canva. E-sign: DocuSign, Dropbox Sign, Adobe Sign. Finance: QuickBooks, Xero, Stripe. Workspace/Docs/Comms: Notion, Slack, Teams. Tasks/PM: Asana, Linear, ClickUp, Trello, Jira. | Unified framework: metadata-first ingest, Jobs/Queue sync, health, import to Vault/Tasks. |
| **Admin, Billing, Metering** | Feature flags; usage metering (docs, signatures, AI/RAG, storage GB, connectors); Stripe subscriptions/trials; plan enforcement (soft nudges → hard caps); admin usage dashboards. | Tax/proration minimal viable; Enterprise later. |
| **Security & Compliance** | Org-scoped RLS; least-privilege; webhook idempotency + DLQ/replay; incident + backup/restore runbooks (tested); WCAG AA sweep; DPA template + security/architecture whitepaper; documented PII model with export/erasure. | Security runbooks are release-gated. |
